# Tutorial V3: карта формул из GoogleTest.xlsx (`long` / `short`)

Документ фиксирует логику расчета позиции, стопа и частичных выходов из Google Sheets,
чтобы потом перенести в Python-модуль без потери деталей.

## Что обнаружено в файле

- В локальной книге `GoogleTest.xlsx` **нет** листа `long_short_database`.
- Есть листы `long` и `short` с одинаковой структурой колонок и почти одинаковыми формулами.
- Ключевое отличие long/short: знак смещения относительно базовой цены (`+` для long, `-` для short).

## Важные колонки и формулы (строка 2 как шаблон)

Ниже перечислены расчетные колонки, которые протянуты вниз по строкам.

- `A`: идентификатор строки сигнала  
  - long: `=CONCATENATE("long ",ROW()-2)`  
  - short: `=CONCATENATE("short ",ROW()-2)`

- `C`: номер недели  
  - `=WEEKNUM(B2)`

- `I`: день недели из даты  
  - `=TEXT(B2, "DDD")`

### Блок размера позиции

- `S`: требуемый номинал позиции (USDT) под риск
  - long: `=100*O2/(100-(AB2*100/Y2))`
  - short: `=100*O2/((AB2*100/Y2)-100)`

- `T`: номинал позиции в USDT по цене входа
  - `=U2*Y2`

- `U`: количество монет (округление шагом `V`)
  - `=MROUND(S2/Y2,V2)`

### Блок цены входа / стопа / TP

- `Y`: цена входа (смещение от базы `K` на `X`)
  - long: `=MROUND(K2+X2,W2)`
  - short: `=MROUND(K2-X2,W2)`

- `Z`: "общий объем" = `U`
  - `=U2`

- `AA`: TP1
  - long: `=MROUND((Y2+M2*Q2),W2)`
  - short: `=MROUND((Y2-M2*Q2),W2)`

- `AB`: стоп-лосс
  - long: `=MROUND(Y2-M2,W2)`
  - short: `=MROUND(Y2+M2,W2)`

### Блок частичных выходов (3 этапа)

#### Выход 1
- `AC`: объем на TP1
  - `=MROUND(U2*AD2/100,V2)`
- `AE`: R:R для TP1
  - `=(AA2-Y2)/(Y2-AB2)`
- `AH`: прибыль к риску (TP1)
  - `=AG2/O2`

#### Выход 2
- `AK`: объем на TP2
  - `=MROUND(((AJ2*Z2)/100),V2)`
- `AM`: цена TP2
  - long: `=MROUND((Y2+M2*AL2),W2)`
  - short: `=MROUND((Y2-M2*AL2),W2)`
- `AO`: R:R для TP2
  - `=(Y2-AM2)/(AB2-Y2)`
- `AQ`: прибыль к риску (TP2)
  - `=AP2/O2`

#### Выход 3
- `AT`: остаток объема после TP1/TP2
  - `=Z2-AC2-AK2`
- `AV`: цена TP3
  - long: `=MROUND((Y2+M2*AU2),W2)`
  - short: `=MROUND((Y2-M2*AU2),W2)`
- `AZ`: прибыль к риску (TP3)
  - `=AY2/O2`

### Итог PnL по сценарию

- `BB`: итоговая прибыль (USDT), ветвление по статусам частичных выходов
  - `=IF(AF2="не ...",0,IF(AF2="да",AI2,AG2+IF(AN2="да",AP2,AR2)+IF(AW2="да",AY2,BA2)))`
  - В файле строки статусов на русском; в коде лучше нормализовать в enum.

- `BC`: прибыль/риск (итог)
  - `=BB2/O2`

## Что важно перенести в код 1:1

- Округление:
  - по цене: `MROUND(..., W)` (тик цены)
  - по объему: `MROUND(..., V)` (шаг количества)
- Разделение логики long/short только в местах со знаком.
- Сценарный расчет итога через статусы частичных выходов.
- Обязательные защитные проверки:
  - деление на ноль (`O`, `Y`, `Y-AB`, `AB-Y`);
  - отрицательный или нулевой остаток объема после TP1/TP2.

## Задаваемые параметры (входные, не вычисляются формулами)

Ниже то, что должно приходить извне (из конфига, сигнала или ручного ввода), а не считаться
внутри расчетчика.

### Параметры риска и входа

- `risk_usdt` (`O`) — риск на сделку в USDT.
- `base_price` (`K`) — базовая цена, от которой строится вход.
- `entry_offset` (`X`) — люфт входа (абсолютный сдвиг от `base_price`).
- `entry_offset_pct` (опционально) — если в стратегии задается в %, перед расчетом переводить в абсолют:
  `entry_offset = base_price * entry_offset_pct / 100`.

### Параметры стопа

- `atr_value` (`M`) — ATR в цене инструмента.
- `stop_atr_mult` (в формулах эквивалент через `M`) — множитель ATR для стопа.
- `stop_loss_pct` (опционально, альтернативный режим) — если стоп задается в процентах:
  - long: `stop_price = entry_price * (1 - stop_loss_pct/100)`
  - short: `stop_price = entry_price * (1 + stop_loss_pct/100)`
  Важно выбрать **один** режим: ATR-стоп или %-стоп.

### Параметры тейков

- `tp1_atr_mult` (`Q`) — расстояние TP1 в ATR.
- `tp2_atr_mult` (`AL`) — расстояние TP2 в ATR.
- `tp3_atr_mult` (`AU`) — расстояние TP3 в ATR.
- `tp1_share_pct` (`AD`) — доля объема на TP1.
- `tp2_share_pct` (`AJ`) — доля объема на TP2.
- `tp3_share_pct` — остаток, обычно `100 - tp1_share_pct - tp2_share_pct`.

### Параметры исполнения/округления

- `price_tick` (`W`) — шаг цены (тик), для `MROUND` по ценам.
- `qty_step` (`V`) — шаг количества, для `MROUND` по объему.
- `side` (`long`/`short`) — направление, определяет знак в формулах.
- `symbol` — инструмент.

## Что считается автоматически (выход расчетчика)

- `entry_price` (`Y`)
- `stop_price` (`AB`)
- `qty_total` (`U`)
- `tp1_price/tp2_price/tp3_price` (`AA`/`AM`/`AV`)
- `tp1_qty/tp2_qty/tp3_qty` (`AC`/`AK`/`AT`)
- `rr_tp1/rr_tp2/rr_total` (`AE`/`AO`/`BC`)
- `pnl_total_usdt` (`BB`) при заданных статусах исполнения TP.
