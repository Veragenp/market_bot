# 🧪 ТЕСТОВЫЙ КОНТУР — Руководство

## 📋 Обзор

Тестовый контур позволяет быстро проверять весь торговый цикл **без ожидания реальных движений цен**. Вместо сложных алгоритмов поиска уровней используется упрощённая генерация:

- **LONG уровень**: `current_price - offset * ATR`
- **SHORT уровень**: `current_price + offset * ATR`

Это позволяет тестировать:
- Entry detector (мониторинг пересечений)
- Открытие позиций
- Закрытие позиций (TP/SL)
- rebuild противоположной стороны
- Полный цикл от arming → in_position → closed

---

## ⚙️ Настройка

### Включение тестового режима

В файле `.env` (или `trading_bot/.env`):

```env
# Включить тестовый режим
TEST_MODE=1

# Смещение уровней от текущей цены в ATR (по умолчанию 0.2)
# LONG: current_price - 0.2 * ATR
# SHORT: current_price + 0.2 * ATR
TEST_LEVEL_OFFSET_ATR=0.2

# Смещение для rebuild противоположной стороны (по умолчанию 0.4)
# После входа в LONG: SHORT = current_price + 0.4 * ATR
TEST_OPPOSITE_OFFSET_ATR=0.4

# Количество символов для тестового цикла (по умолчанию 10)
TEST_CYCLE_SYMBOLS_COUNT=10

# Интервал пересоздания уровней (сек) - по умолчанию 60
TEST_CYCLE_INTERVAL_SEC=60
```

### Отключение

```env
TEST_MODE=0
```

## 📊 Что происходит в тестовом режиме

### 1. Генерация уровней

**Файл**: `trading_bot/analytics/test_level_generator.py`

При каждом запуске `structural` этапа:
1. Сбрасывается старый тестовый цикл
2. Выбираются первые N символов из `TRADING_SYMBOLS`
3. Для каждого символа:
   - Получается текущая цена (из price_feed)
   - Получается ATR (из instruments)
   - Создаются LONG и SHORT уровни
4. Записываются в БД:
   - `structural_cycles`
   - `structural_cycle_symbols`
   - `cycle_levels`
5. Обновляется `trading_state`

**Пример лога:**
```
TEST_MODE: Selected 10 symbols for test cycle
TEST_MODE: BTC/USDT - current=68500.00 LONG=68400.00 SHORT=68600.00 ATR=500.00
TEST_MODE: ETH/USDT - current=3500.00 LONG=3480.00 SHORT=3520.00 ATR=100.00
TEST_MODE: Generated 20 levels for 10 symbols (cycle=a1b2c3d4)
```

### 2. Entry detector

Работает **как в продакшене**:
- Мониторит пересечения уровней
- Проверяет ATR-порог (ENTRY_GATE_LONG_ATR_THRESHOLD_PCT)
- Выставляет лимитные ордера на биржу

**Важно**: В тестовом режиме уровни находятся близко к цене (0.2 ATR), поэтому срабатывания происходят быстро.

### 3. Rebuild противоположной стороны

**Файл**: `trading_bot/analytics/structural_cycle.py` → `_rebuild_side_test_mode()`

После входа в LONG:
- SHORT уровень перестраивается на `current_price + 0.4 * ATR`

После входа в SHORT:
- LONG уровень перестраивается на `current_price - 0.4 * ATR`

## 📈 Анализ тестового прогона

### Анализатор логов

**Файл**: `trading_bot/scripts/analyze_test_run.py`

**Запуск:**
```powershell
# Вывод в консоль
python -m trading_bot.scripts.analyze_test_run

# Сохранение в файл (UTF-8)
python -m trading_bot.scripts.analyze_test_run --output report.txt

# Автоматическое сохранение с датой
python -m trading_bot.scripts.analyze_test_run --format file

# Режим наблюдения (обновление каждые 10 секунд)
python -m trading_bot.scripts.analyze_test_run --watch 10
```

**Аргументы:**
- `--db PATH` — путь к базе данных (по умолчанию: `data/market_data.db`)
- `--logs DIR` — путь к директории с логами (по умолчанию: `logs/`)
- `--output FILE` — сохранить отчёт в файл
- `--format console|file` — формат вывода (console — в консоль, file — в файл)
- `--watch N` — режим наблюдения с интервалом N секунд

1. Общая информация (режим, время, символы)

3. Позиции (открытые/закрытые, PnL, причины закрытия)
4. Ордера (статусы, типы)
5. Ошибки и предупреждения

## ⚠️ Важные замечания

### 1. Тестовый режим ≠ Продакшен

- Уровни **искусственные** (не реальные HVN/VP)
- Нет сложной логики structural cycle (W*, медиана, pool)
- Нет ожидания реальных движений цен

### 2. BYBIT_USE_DEMO

Рекомендуется использовать **Demo mode** биржи:

```env
BYBIT_USE_DEMO=1
```

Это защитит реальные средства от случайных ордеров.

### 3. BYBIT_EXECUTION_ENABLED

Для тестирования открытия позиций:

```env
BYBIT_EXECUTION_ENABLED=1
```

Если выключено — ордера не выставляются (только логирование).

### 4. Ограничения

- **Не тестирует**: качество уровней, выбор pool, корреляцию
- **Тестирует**: полный цикл входа/выхода, работу entry detector, PnL математика

## 🧪 Примеры тестовых сценариев

### Сценарий 1: Быстрая проверка entry detector

```env
TEST_MODE=1
TEST_LEVEL_OFFSET_ATR=0.2  # Уровни близко
ENTRY_GATE_LONG_ATR_THRESHOLD_PCT=0.5  # Низкий порог
```

**Ожидание**: Быстрые срабатывания (5-10 минут)

### Сценарий 2: Проверка TP/SL

```env
TEST_MODE=1
TEST_LEVEL_OFFSET_ATR=0.2
POSITION_TP1_ATR_MULT=3  # Стандартные значения
POSITION_STOP_ATR_MULT=1
```

**Ожидание**: Позиции закрываются по TP/SL через 15-30 минут

### Сценарий 3: Проверка rebuild opposite

```env
TEST_MODE=1
TEST_OPPOSITE_OFFSET_ATR=0.4  # Удалённая противоположная сторона
```

**Ожидание**: После входа LONG — SHORT уровень на +0.4 ATR

### Сценарий 4: Full cycle (arm → in_pos → close → arm)

```env
TEST_MODE=1
TEST_CYCLE_SYMBOLS_COUNT=10  # Больше символов = больше шансов на срабатывание
```

**Ожидание**: Полный цикл с закрытием и новым армингом

## 🔧 Отладка

### Проверка тестовых уровней

```sql
-- Текущий цикл
SELECT * FROM trading_state WHERE id=1;

-- Уровни
SELECT cycle_id, symbol, direction, level_price 
FROM cycle_levels 
WHERE level_type='test' 
ORDER BY created_at DESC LIMIT 20;

-- Structural symbols
SELECT * FROM structural_cycle_symbols 
WHERE ref_price_source='test' 
ORDER BY created_at DESC LIMIT 10;
```
