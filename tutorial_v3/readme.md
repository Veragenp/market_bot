Проект представляет собой систему автоматизированной торговли на криптобирже Bybit, интегрированную с Google Sheets и Telegram для управления сделками и мониторинга рынка. Основная задача — мониторить цены криптовалют, генерировать сигналы на вход в сделки (LONG/SHORT) на основе пересечения ценовых уровней, и управлять ордерами с учетом данных из Google Sheets. Вот как работают ключевые компоненты:

1. BybitAPI (bybit_api.py)
   Синглтон-класс для взаимодействия с API Bybit.  
   Функционал:  
   Получение исторических данных (цены high/low за 7 дней, объем торгов за 24 часа).

Запрос информации об инструментах (размер тика, минимальный объем).

Получение комиссий (maker/taker).

Управление ордерами: размещение лимитных ордеров, проверка статуса, отмена.

Подписка на WebSocket для получения текущих цен в реальном времени.

Особенности:  
Использует библиотеку pybit для HTTP и WebSocket-запросов.

Логирует все действия и ошибки.

Реализует паттерн Singleton, чтобы избежать множественных подключений к API.

2. GoogleSheetsClient (google_sheets.py)
   Синглтон-класс для работы с Google Sheets.  
   Функционал:  
   Чтение данных из листов (database, analitics, long, short).

Получение списка монет для торговли (из листа analitics, столбец D = TRUE).

Получение данных о сделках (из листов long и short, столбец F = TRUE).

Обновление ячеек (например, статус сделки, сброс столбцов).

Пакетное обновление данных с повторными попытками при ошибке 503.

Особенности:  
Использует gspread для работы с Google Sheets API.

Логирует действия и обрабатывает ошибки (например, отсутствие листа).

Хранит данные о монетах и уровнях входа (long_level, short_level).

3. PriceFetcher (fetch_prices.py)
   Класс для получения и мониторинга текущих цен через WebSocket Bybit.  
   Функционал:  
   Инициализируется с экземпляром BybitAPI.

Получает список монет для мониторинга из Google Sheets (analitics).

Проверяет валидность символов через REST API.

Подписывается на WebSocket для получения цен в реальном времени.

Хранит текущие цены в словаре current_prices.

Логирует цены каждые 10 секунд.

Особенности:  
Реализует механизм переподключения WebSocket при сбоях.

Работает в бесконечном цикле, пока не остановлен.

Проверяет валидность символов перед подпиской.

4. TradingMonitor (traiding_monitor.py)
   Класс для мониторинга цен и генерации сигналов на основе пересечения уровней.  
   Функционал:  
   Получает список монет и уровни (long_level, short_level) из Google Sheets.

Сравнивает текущие цены (из PriceFetcher) с уровнями.

Регистрирует алерты при пересечении уровней (цена падает ниже long_level или растет выше short_level).

Отслеживает количество алертов в окне времени (ALERT_TIMEOUT_MINUTES).

Генерирует сигналы для входа в сделку (LONG или SHORT), если выполнены условия:  
Минимум MIN_ALERTS_COUNT алертов.

Прошло ALERT_TIMEOUT_MINUTES минут.

Генерирует сигналы отмены (CANCEL_LONG, CANCEL_SHORT), если в окне времени зафиксировано MAX_ADDITIONAL_ALERTS дополнительных алертов.

Особенности:  
Работает в отдельном потоке, запуская PriceFetcher.

Передает сигналы в TradeSignalProcessor для дальнейшей обработки.

Учитывает флаги allow_long_signals и allow_short_signals из Main.

5. TradeSignalProcessor (trade_signal_processor.py)
   Класс для обработки сигналов от TradingMonitor и управления входом в сделки.  
   Функционал:  
   Обрабатывает сигналы LONG, SHORT, CANCEL_LONG, CANCEL_SHORT.

Для CANCEL_LONG/CANCEL_SHORT: сбрасывает столбец E (Торговля) в соответствующих листах и останавливает программу.

Для LONG/SHORT:  
Получает данные монет с E=TRUE из листов long/short.

Проверяет условия входа с учетом ATR (Average True Range) из листа database:  
Для LONG: цена >= уровень - LONG_ATR_THRESHOLD_PERCENT \* ATR.

Для SHORT: цена < уровень - SHORT_ATR_THRESHOLD_PERCENT \* ATR, и монета не имеет алерта.

Если условия выполнены, проставляет F=TRUE (вход) и сбрасывает E.

Отправляет уведомления в Telegram о входе в сделку.

Особенности:  
Обновляет Google Sheets через пакетные запросы.

Блокирует повторные сигналы, меняя флаги allow_long_signals/allow_short_signals.

6. TradeManager (trade_manager.py)
   Класс для управления сделками на основе данных из Google Sheets.  
   Функционал:  
   Периодически проверяет листы long и short на сделки с F=TRUE.

Для каждой сделки:  
Проверяет наличие стоп-лосса (иначе отменяет).

Отправляет запрос на подтверждение в Telegram.

После подтверждения (yes) размещает лимитный ордер через BybitAPI.

Обновляет статус в Google Sheets (вход, ожидание, ордер, ошибка входа).

Мониторит активные ордера:  
Проверяет статус (Filled, Cancelled, Rejected).

Обновляет статус в Google Sheets и отправляет уведомления.

Отменяет ордера, если достигнут лимит позиций (MAX_TRADES).

Особенности:  
Работает в двух потоках: один для проверки новых сделок, другой для мониторинга ордеров.

Обрабатывает ответы пользователей через Telegram (кнопки или текст).

7. Main (main.py)
   Основной класс, объединяющий все компоненты.  
   Функционал:  
   Инициализирует BybitAPI, GoogleSheetsClient, TelegramClient, TradingMonitor, TradeSignalProcessor, TradeManager.

Сбрасывает столбец F в листах long и short при запуске.

Запрашивает у пользователя через Telegram обновление статических и исторических данных.

Запускает все модули в отдельных потоках.

Управляет остановкой программы.

Особенности:  
Использует конфигурацию из .env (например, ключи API, ID таблицы).

Работает в основном потоке для Telegram polling.

8. populate_static_data.py и populate_historical_data.py
   Скрипты для заполнения Google Sheets данными.  
   populate_static_data.py:  
   Заполняет лист database статическими данными: символы, размер тика, минимальный объем, комиссии.

Получает данные через BybitAPI.

populate_historical_data.py:  
Обновляет лист database историческими данными: цены high/low за 7 дней, объем за 24 часа, ATR.

Фильтрует символы по порогу объема (VOLUME_THRESHOLD).

Особенности:  
Используют GoogleSheetsClient для записи данных.

Выполняют пакетные обновления для оптимизации.

9. TelegramClient (telegram_client.py)
   Синглтон-класс для взаимодействия с Telegram.  
   Функционал:  
   Отправляет сообщения с текстом или кнопками (Да/Нет).

Обрабатывает ответы пользователей (текстовые или через кнопки).

Регистрирует callback-функции для обработки ответов.

Экранирует HTML-символы в сообщениях.

Особенности:  
Использует python-telegram-bot для работы с Telegram API.

Избегает повторной обработки сообщений.

Общий процесс работы
Запуск программы (Main):  
Инициализируются все компоненты.

Сбрасывается столбец F в листах long и short.

Пользователь подтверждает обновление данных (populate_static_data, populate_historical_data).

Мониторинг цен (TradingMonitor + PriceFetcher):  
PriceFetcher получает текущие цены через WebSocket.

TradingMonitor сравнивает цены с уровнями из Google Sheets, регистрирует алерты.

Генерация сигналов (TradingMonitor):  
При достижении MIN_ALERTS_COUNT алертов и истечении ALERT_TIMEOUT_MINUTES генерируется сигнал LONG/SHORT.

При превышении MAX_ADDITIONAL_ALERTS генерируется сигнал отмены.

Обработка сигналов (TradeSignalProcessor):  
Проверяет условия входа с учетом ATR.

Обновляет Google Sheets (проставляет F=TRUE для входа).

Управление сделками (TradeManager):  
Находит сделки с F=TRUE, запрашивает подтверждение через Telegram.

Размещает ордера, мониторит их статус, обновляет Google Sheets.

Взаимодействие с пользователем (TelegramClient):  
Отправляет уведомления и обрабатывает подтверждения/отмены сделок.

Потоки выполнения
Основной поток: TelegramClient (polling для обработки сообщений).

Поток TradingMonitor: запускает PriceFetcher и мониторит цены/сигналы.

Поток TradeSignalProcessor: ожидает сигналы от TradingMonitor.

Поток TradeManager: два подпотока (проверка сделок и мониторинг ордеров).

Источники данных
Google Sheets:  
Лист database: исторические данные, ATR, объем, комиссии.

Лист analitics: список монет для торговли, уровни long_level/short_level.

Листы long/short: данные о сделках (вход, статус, параметры).

Bybit API: текущие цены, исторические данные, информация об инструментах.

Telegram: взаимодействие с пользователем.

С корректировками трейдинг менеджер - ограничение сделок.
